# خريطة التطور المرحلية (Phased Roadmap)

## من AI Kernel الحالي → النظام الكامل وفق الرؤية

هذه الخريطة تحوّل **الرؤية الاستراتيجية** إلى مسار تنفيذي واقعي، تدريجي، وغير مُرهق، مع الحفاظ على جوهر المشروع:

> *Agent لغوي تنفيذي، يتطور فعليًا، ويُدار لغويًا، دون أن يتحول إلى نظام برمجة ثقيل.*

---

## Phase 0 — الحالة الحالية (Baseline)

### الوضع الحالي (AI Kernel Core)

**ما هو موجود فعليًا:**

* Client محلي (Node / CLI / VS Code)
* قدرة قراءة ملفات المشروع تلقائيًا
* Council (نماذج متعددة) للتحليل والتفكير
* توليد اقتراحات (Diff / Logic)
* Test Runner
* Apply / Rollback
* تحكم لغوي بالأوامر (`@start`, `@stop`, `@focus`, `@avoid`)

**ما لا يوجد بعد (بوضوح):**

* Agent Loop دائم
* ذاكرة تعلم حقيقية
* حكم مركزي (Judge)
* Sandboxes مستقلة

> هذه المرحلة مكتملة ذهنيًا، وجاهزة للبناء الفعلي.

---

## Phase 1 — Agent Loop (Minimum Viable Agent)

### الهدف

تحويل AI Kernel من "أداة تُستدعى" إلى **Agent يعمل تلقائيًا عند الطلب**.

### ما يُبنى في هذه المرحلة

* Agent Loop بسيط:

  ```
  Observe → Plan → Generate → Test → Apply → Repeat
  ```

* Auto-Apply مفعّل افتراضيًا
* Checkpoints بعد كل Patch
* أوامر لغوية للتحكم أثناء التشغيل

### ما لا يُبنى (مهم)

* لا ذاكرة طويلة الأمد
* لا تحسين برومبتات ذاتي
* لا حكم مركزي

### ناتج المرحلة

> Agent فعلي يعمل، يُطبّق، ويتوقف عند أمرك.

---

## Phase 2 — Sandboxed Execution (Safety without Friction)

### الهدف

تمكين **التجربة الآمنة السريعة** بدون إدخال تعقيد واجهات.

### ما يُبنى

* Sandboxes مؤقتة (Docker / E2B / FS Copy)
* كل جلسة Agent تعمل في Sandbox
* Apply = دمج النتائج الناجحة فقط
* Rollback تلقائي عند الفشل

### لماذا هذه المرحلة مهمة

* تسمح بالـ Auto-Apply بثقة
* تفتح الباب لتجارب أوسع (Legacy / Refactor)

### ناتج المرحلة

> Agent قوي، سريع، وآمن بدون أن تشعر بالحوكمة.

---

## Phase 3 — Judge Agent (العقل الحاكم غير المرئي)

### الهدف

إضافة **طبقة حكم ذكية** دون تحويل النظام إلى بيروقراطية.

### ما هو Judge Agent

* ليس واجهة
* ليس Controller
* بل **قرار داخلي**:

  * يقارن نتائج النماذج
  * يحسم التعارض
  * يقرر متى يعيد التخطيط

### ما يُبنى

* Scoring لكل Patch (أداء / مخاطرة / تأثير)
* Consensus Threshold
* آلية Re-plan تلقائي عند التعارض العالي

### ما لا يُبنى

* لا تدخل يدوي
* لا إعدادات للمستخدم

### ناتج المرحلة

> قرارات أعلى جودة بدون أي عبء عليك.

---

## Phase 4 — Learning & Memory (التطور الحقيقي)

### الهدف

جعل النظام **يتحسن مع الاستخدام** لا مع التحديثات اليدوية.

### ما يُبنى

* Memory Store (Vector DB)
* تسجيل:

  * ما تم قبوله
  * ما تم رفضه
  * أسباب الإيقاف
* Negative Memory للأخطاء المتكررة
* تحسين التخطيط بناءً على التجارب السابقة

### ما لا يُبنى

* لا تعديل ذاتي للكود
* لا تغيير قواعد التشغيل

### ناتج المرحلة

> Agent يتعلم ذوقك التقني دون أن تطلب ذلك.

---

## Phase 5 — Prompt & Strategy Self-Refinement

### الهدف

تحسين **طريقة التفكير** لا مجرد التنفيذ.

### ما يُبنى

* تعديل داخلي للـ prompts بناءً على:

  * الفشل المتكرر
  * نجاح الأنماط
* تحسين توزيع المهام داخل المجلس
* تقليل الضجيج وزيادة الدقة

### الضبط الحرج

* التعديل لا يتم أثناء الجلسة
* يتم بين الجلسات فقط

### ناتج المرحلة

> ذكاء أعلى بنفس الواجهة البسيطة.

---

## Phase 6 — Legacy Modernization Engine

### الهدف

تحويل النظام إلى أداة **إحياء أنظمة قديمة**.

### ما يُبنى

* قراءة أنظمة Legacy بدون Tests
* توليد Tests تلقائيًا
* Modernization تدريجي
* توثيق تلقائي أثناء التحسين

### لماذا هذه المرحلة قاتلة

* قيمة سوقية عالية
* مشكلة حقيقية
* لا يوجد حل جيد حاليًا

### ناتج المرحلة

> Agent لا يكتب كودًا فقط… بل ينقذ أنظمة.

---

## Phase 7 — Multi-Agent Collaboration (اختياري)

### الهدف

فتح المجال لتعاون عدة Agents على نفس النظام.

### ما يُبنى

* Agents متخصصة (Security / Performance / Refactor)
* Judge أعلى
* تناوب منضبط

### ملاحظة

هذه المرحلة **غير ضرورية** للنجاح الأولي.

---

## الخلاصة التنفيذية

* كل Phase:

  * مستقلة
  * قابلة للإطلاق
  * تضيف قيمة حقيقية
* لا قفزات خطيرة
* لا إعادة بناء

> **يمكن إيقاف التطوير عند أي Phase، والنظام سيبقى مفيدًا وقابلًا للاستخدام.**

---

## التوصية النهائية

إذا أردت بناء هذا المشروع بذكاء:

* ابدأ بـ Phase 1 + Phase 2 فقط
* أثبت القيمة
* ثم انتقل تدريجيًا

هذه الخريطة تحميك من:

* الإفراط الهندسي
* تضخم المشروع
* فقدان جوهر الرؤية
